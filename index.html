Detect Transfers

Finds pairs where money left one bank account and an equal amount arrived in another within a few days — i.e., transfers between your accounts (Chase → Metro, intercompany wires, owner loans).

Why it matters: a real transfer isn't income or an expense. Confirming a pair tags both rows with is_transfer = true so they drop out of P&L.

Exact match only: amounts must equal to the cent. Pairs at the same merchant (e.g. both legs say "City Farmers Market") are auto-excluded — those are refunds or coincidences, not transfers. Only suggestions scoring ≥ 0.7 are shown.

Import Journal Entries

Paste CSV or upload a file. Each row is one line of a journal entry. Rows that share the same Date + Memo are grouped into the same entry. Each entry must balance (debits = credits). Accounts can be referenced by code (e.g. 1000) or name (e.g. Cash on Hand) or full code name.
Sample format 
Date,Memo,Account,Description,Debit,Credit
2025-01-01,Opening balances,1000,,50000,
2025-01-01,Opening balances,3000,Owner contribution,,50000
2025-01-15,January rent,6700 Rent,Suite 101,3000,
2025-01-15,January rent,1000,Payment,,3000
